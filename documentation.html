s-found 2 Game Functions Documentation

This document describes a set of JavaScript functions used in a s-found 2-based game for sprite management, collision detection, movement, and other game mechanics.

onSpriteClick(sprite, pointer)

Parameters:

  • sprite: The sprite object that was clicked.
  • pointer: The pointer object representing the click event.

Description: A default event handler for sprite click events. Can be overridden to implement custom behavior.

RadianstoDegrees(angle)

Parameters:

  • angle: Angle in radians.

Description: Converts an angle from radians to degrees.

Returns: The angle in degrees.

DegreestoRadians(angle)

Parameters:

  • angle: Angle in degrees.

Description: Converts an angle from degrees to radians.

Returns: The angle in radians.

normalize_angle_hack(angle)

Parameters:

  • angle: Angle in degrees to normalize.

Description: Normalizes an angle to the range [0, 360) degrees using a repetitive modulo operation.

Returns: The normalized angle in degrees.

clone_sprite()

Parameters: None

Description: Creates a clone of the current sprite, copying its position, key, frame, and frames. The clone is added to the front layer, scaled, and physics-enabled with centered rotation. The original sprite is hidden to avoid duplication.

destroy_current_sprite()

Parameters: None

Description: Destroys the current sprite and its frames. If the sprite is a clone, it removes the clone from the sprite's clone list.

hide_current_sprite()

Parameters: None

Description: Hides the current sprite and its frames by setting their visibility to false. Handles both cloned and non-cloned sprites.

show_current_sprite()

Parameters: None

Description: Shows the current sprite and its current frame by setting their visibility to true. Handles both cloned and non-cloned sprites.

find_sprite_object_by_name_new(sprite_name)

Parameters:

  • sprite_name: Name of the sprite to find.

Description: Similar to find_sprite_object_by_name, finds a sprite by name, handling clones if applicable.

Returns: The sprite object or null if not found or if no sprite name is provided.

find_target_sprite_with_clones_object_by_name(sprite_name)

Parameters:

  • sprite_name: Name of the target sprite to find.

Description: Finds a target sprite by name, returning a clone if a target clone is specified via current_target_clone.

Returns: The sprite object or null if not found or if no sprite name is provided.

set_current_sprite_name(sprite_name)

Parameters:

  • sprite_name: Name of the sprite to set as current.

Description: Sets the global current_sprite_name to the provided sprite name.

set_target_sprite_name(sprite_name)

Parameters:

  • sprite_name: Name of the sprite to set as target.

Description: Sets the global target_sprite_name to the provided sprite name.

point_in_direction_degrees(angle_degrees)

Parameters:

  • angle_degrees: Angle in degrees to rotate the sprite.

Description: Rotates the current sprite to face the specified angle (in degrees).

point_sprite_in_direction_to_point(destination_x, destination_y)

Parameters:

  • destination_x: X-coordinate of the target point.
  • destination_y: Y-coordinate of the target point.

Description: Rotates the current sprite to face a specified point in the game world.

point_in_direction(x1, y1, x2, y2)

Parameters:

  • x1, y1: Coordinates of the starting point.
  • x2, y2: Coordinates of the target point.

Description: Calculates the angle (in degrees) between two points using Math.atan2.

Returns: The angle in degrees.

point_spriteA_in_direction_spriteB(a_x, a_y, b_x, b_y)

Parameters:

  • a_x, a_y: Coordinates of the current sprite (ignored, as sprite is fetched by name).
  • b_x, b_y: Coordinates of the target sprite (ignored, as sprite is fetched by name).

Description: Rotates the current sprite to face the target sprite using their current positions.

foward(distance)

Parameters:

  • distance: Distance to move the sprite forward.

Description: Moves the current sprite forward by the specified distance in the direction it is facing, updating frame positions accordingly.

distance_between_sprite_and_target()

Parameters: None

Description: Calculates the Euclidean distance between the current sprite and the target sprite.

Returns: The distance as a number.

set_x_to(X)

Parameters:

  • X: The x-coordinate to set for the sprite.

Description: Sets the x-coordinate of the current sprite and updates its frame positions.

set_y_to(Y)

Parameters:

  • Y: The y-coordinate to set for the sprite.

Description: Sets the y-coordinate of the current sprite and updates its frame positions.

turn_right(degrees)

Parameters:

  • degrees: Number of degrees to rotate the sprite clockwise.

Description: Rotates the current sprite clockwise by the specified degrees and updates frame positions.

turn_left(degrees)

Parameters:

  • degrees: Number of degrees to rotate the sprite counterclockwise.

Description: Rotates the current sprite counterclockwise by the specified degrees and updates frame positions.

get_x_value()

Parameters: None

Description: Retrieves the x-coordinate of the current sprite.

Returns: The x-coordinate as a number.

get_y_value()

Parameters: None

Description: Retrieves the y-coordinate of the current sprite.

Returns: The y-coordinate as a number.

sprite_width()

Parameters: None

Description: Retrieves the width of the current sprite.

Returns: The width as a number.

sprite_height()

Parameters: None

Description: Retrieves the height of the current sprite.

Returns: The height as a number.

biased(a, b, c, d)

Parameters:

  • a, b: Range boundaries for the first interval.
  • c, d: Range boundaries for the second interval.

Description: Checks if the point c lies within the range [a, b) or if d lies within (a, b].

Returns: Boolean indicating if the condition is met.

unbiased(a, b, c, d)

Parameters:

  • a, b: Range boundaries for the first interval.
  • c, d: Range boundaries for the second interval.

Description: Checks if either interval overlaps the other using the biased function.

Returns: Boolean indicating overlap.

collision(r1, r2)

Parameters:

  • r1, r2: Rectangle objects with top, bottom, left, right properties.

Description: Checks if two rectangles overlap using the unbiased function for their x and y ranges.

Returns: Boolean indicating if the rectangles collide.

intersectRect(s1, s2)

Parameters:

  • s1, s2: Sprite objects with x, y, width, height properties.

Description: Converts sprite coordinates and dimensions into rectangle objects and checks for collision using the collision function.

Returns: Boolean indicating if the sprites' bounding boxes intersect.

current_sprite_clones_are_colliding_with_target_sprite_clones()

Parameters: None

Description: Checks if the current sprite (or its clone) collides with the target sprite (or its clone) using bounding box collision detection.

Returns: Boolean indicating collision.

is_colliding_with_target()

Parameters: None

Description: Checks if the current sprite collides with the target sprite using bounding box collision detection.

Returns: Boolean indicating collision.

next_fame()

Parameters: None

Description: Advances the current sprite to the next frame in its animation, looping back to the first frame if at the end. Updates visibility based on the sprite's hidden state.

set_frame_to_frame_number(frame_number_or_alias, force_game_object)

Parameters:

  • frame_number_or_alias: Frame number or alias to set.
  • force_game_object: Optional sprite name to override the current sprite.

Description: Sets the sprite's frame to the specified number or alias, handling both cloned and non-cloned sprites. Updates visibility accordingly.

current_frame_number()

Parameters: None

Description: Retrieves the current frame number of the sprite. Hides the current frame before returning the number.

Returns: The current frame number or 0 if no frames exist.

update_position_of_frames(game_Object)

Parameters:

  • game_Object: The sprite object whose frames need updating.

Description: Updates the position and rotation of all frames of a sprite to match the main sprite's position and angle.

loadAndPlaySound(key, url)

Parameters:

  • key: The key for the audio asset.
  • url: The URL of the audio file to load.

Description: Loads an audio file if not already cached and plays it using Phaser's audio system.
